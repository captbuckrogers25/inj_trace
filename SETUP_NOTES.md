# inj_trace — Setup & Build Notes

Summary of work done to create and configure this library, including fixes to
upstream dependencies. Intended as a reference for anyone reproducing this
environment.

---

## 1. What was built

`inj_trace` is a Python library wrapping two pre-existing local packages:

- **LANLGeoMag / lgmpy** — C library with Python ctypes bindings for empirical
  magnetic field models (T89, TS04, OP77), L-shell (L*) calculations, field-line
  tracing, and coordinate transforms. Installed from `~/LANLGeoMag`.
- **SHIELDS-PTM / ptm_python** — Fortran guiding-centre/full-orbit particle
  tracer with Python pre/post-processing utilities. Located at `~/SHIELDS-PTM`.

The library provides an end-to-end workflow:

```
LANLGeoMag field models → PTM field files → PTM particle trace → flux maps + visualization
```

### Library layout

```
inj_trace/
├── config.py              path config, sys.path injection
├── fields/
│   ├── models.py          eval_t89(), eval_ts04(), eval_op77()
│   ├── grid.py            FieldGrid (evaluate B on 3-D GSM grid, parallel)
│   └── writer.py          PTMFieldWriter (write ptm_fields_XXXX.dat, tgrid.dat)
├── runner/
│   ├── ptm_setup.py       PTMRunConfig (wraps ptm_input_creator)
│   └── executor.py        PTMExecutor (subprocess launch, parallel runs)
├── postprocess/
│   ├── trajectories.py    TrajectoryData (load PTM output, compute L*)
│   └── fluxmap.py         FluxMapResult (wraps ptm_postprocessor)
├── visualization/
│   ├── equatorial.py      plot_equatorial_bfield(), plot_equatorial_flux()
│   ├── lshell.py          plot_lshell_map(), plot_lshell_energy_map()
│   ├── timeseries.py      plot_flux_timeseries()
│   ├── trajectories3d.py  plot_trajectory_3d()
│   └── animation.py       animate_equatorial_flux(), animate_lshell()
└── cli/
    ├── config_cmd.py      inj-config set/validate/show
    ├── make_fields.py     inj-make-fields
    ├── run_ptm.py         inj-run
    └── plot.py            inj-plot
```

## 2. LANLGeoMag build

The LANLGeoMag shared library was not pre-built. It was compiled from source.

### Build steps (from `~/LANLGeoMag`)

```bash
cd ~/LANLGeoMag
aclocal && autoheader && autoconf && automake --add-missing
```

`autoreconf -i` does not work; the four commands above must be run separately.

`PKG_CHECK_MODULES` was undefined (no `pkg-config` and no sudo). Fix: wrote a
stub `m4/pkg.m4` that always executes the not-found branch.

```bash
./configure \
  CFLAGS="-I/usr/include/hdf5/serial" \
  LDFLAGS="-L/usr/lib/x86_64-linux-gnu/hdf5/serial"
make
```

Python bindings (there is no top-level `make python` target — must build inside
the `Python/` subdirectory):

```bash
cd ~/LANLGeoMag/Python
make && make install
```

This installs lgmpy to `~/.local/lib/python3.12/site-packages/lgmpy/` and the
shared library to `~/.local/lib/libLanlGeoMag.so`.

## 3. Fixes to lgmpy's generated ctypes wrapper

`lgmpy/Lgm_Wrap.py` is auto-generated by ctypesgen during the Python build.
ctypesgen failed to parse several large or complex structs, leaving function
bindings and struct definitions missing. All fixes were appended to the end of:

```
~/.local/lib/python3.12/site-packages/lgmpy/Lgm_Wrap.py
```

### 3a. Three missing function bindings (opaque struct pointers)

`Lgm_LstarInfo` and `Lgm_MagEphemInfo` were too complex for ctypesgen. Three
functions that take pointers to these were missing. Fix: bind them with
`c_void_p` for the struct-pointer arguments.

```python
Lgm_SetLstarTolerances.argtypes = [c_int, c_int, c_void_p]
Lgm_FreeMagEphemInfo_Children.argtypes = [c_void_p]
Lgm_ComputeLstarVersusPA.argtypes = [c_long, c_double, POINTER(Lgm_Vector),
                                      c_int, POINTER(c_double), c_int, c_void_p]
```

### 3b. `Lgm_QinDentonOne` struct + two functions

ctypesgen skipped the entire `Lgm_QinDentonOne` struct (solar-wind/geomagnetic
index data container). The struct was reconstructed from `Lgm_QinDenton.h` and
defined as a ctypes `Structure`:

```python
class struct_Lgm_QinDentonOne(Structure):
    _fields_ = [('nPnts', c_int), ('Persistence', c_int), ...]
Lgm_QinDentonOne = struct_Lgm_QinDentonOne
```

Two dependent functions were then bound: `Lgm_get_QinDenton_at_JD` and
`Lgm_set_QinDenton`.

### 3c. `Lgm_LstarInfo` struct (16 MB inline-array struct)

`Lgm_LstarInfo` embeds multi-megabyte 2-D arrays directly (e.g.
`s_gsm[400][1000]`). ctypesgen cannot parse it. The struct is 16,356,352 bytes.

All field offsets were determined by compiling a small C program using
`offsetof()` and `sizeof()`. The struct was then defined in ctypes with explicit
padding arrays where needed:

```
nFLsInDriftShell … SaveShellLines   offsets 0–68
nFieldPnts[400]                      offset 68
4-byte alignment gap
s_gsm[400][1000]                     offset 1672
Bmag, x_gsm, y_gsm, z_gsm           (each 3,200,000 bytes)
nPnts                                offset 16001672
MLT, mlat, footprint arrays …
I[400], Bmin[400], Pmin[400] …
DriftOrbitType, nMinima[300] …
VerbosityLevel, PreStr, PostStr
LS, LS_dip_approx, LS_McIlwain_M
xma, yma, ym2, MLATarr …
nBounceRegions, nParticles
Particles[5000]                      offset 16236352
```

Key array types for Python ndarray interop:
- 2-D: `(c_double * 1000) * 400` — supports `arr[i][j]` and `buffer=`
- 1-D: `c_double * 400`, `Lgm_Vector * 400`

### 3d. `Lgm_MagEphemInfo` struct (768 bytes, all-pointer fields)

`Lgm_MagEphemInfo` holds a `Lgm_LstarInfo *` pointer plus dynamically-allocated
arrays for pitch-angle data. Offsets again determined via `offsetof()` in C.
Defined as a 768-byte ctypes `Structure` with:

- `LstarInfo`: `POINTER(struct_Lgm_LstarInfo)` — allows `.contents.VerbosityLevel` etc.
- `Alpha`, `Bm`, `I`, `Lstar`, etc.: `POINTER(c_double)` — allows `[i]` indexing
- `nShellPoints`: `POINTER(c_int)` — not `c_void_p`; Python assigns `[i]`
- Multi-dim pointer fields (`ShellXxx`, `s_gsm`, etc.): `c_void_p`

`Lgm_InitMagEphemInfoDefaults` was also bound.

### 3e. `Lstar` function

The core `int Lstar(Lgm_Vector *, Lgm_LstarInfo *)` function was missing.
Added with the now-defined struct pointer type:

```python
Lstar.argtypes = [POINTER(Lgm_Vector), POINTER(struct_Lgm_LstarInfo)]
Lstar.restype = c_int
```

## 4. Fix to inj_trace config path injection

`config.py::_inject_paths()` originally used `sys.path.insert(0, lgmpy_path)`.
This placed `~/LANLGeoMag/Python` before site-packages, causing Python to find
the source `lgmpy/` directory (which has no `Lgm_Wrap.py`) instead of the
installed version.

**Fix:** `lgmpy_path` is now appended (`sys.path.append`) so the installed
site-packages version always wins. `ptm_python_path` is still inserted at
position 0 (it is not installed, so there is no shadowing risk).

---

## 5. Fix to pyproject.toml build backend

The original `build-backend` was `"setuptools.backends.legacy:build"`, which
requires setuptools ≥ 64 with the backends submodule. The system setuptools
(82.0.1) does not expose that path. Changed to:

```toml
build-backend = "setuptools.build_meta"
```

## 8. Key facts for future reference

| Item | Value |
|------|-------|
| Python version | 3.12 (`/bin/python3`) |
| lgmpy location | `~/.local/lib/python3.12/site-packages/lgmpy/` |
| libLanlGeoMag.so | `~/.local/lib/libLanlGeoMag.so` |
| ptm_python_path | `~/SHIELDS-PTM` (parent of `ptm_python/`, NOT `ptm_python/` itself) |
| ptm executable | `~/SHIELDS-PTM/ptm` |
| Config file | `~/.inj_trace.cfg` (INI format, section `[paths]`) |
| lgmpy not thread-safe | Use `ProcessPoolExecutor` for parallel field / L* evaluation |
| PTM needs ≥ 2 snapshots | `PTMFieldWriter.write_static()` duplicates a single snapshot |
| PTM reads from CWD | `PTMExecutor` `os.chdir`s to `run_dir` before invoking `./ptm` |
